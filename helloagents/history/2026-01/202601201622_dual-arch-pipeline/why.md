# 变更提案: 流水线双架构构建

## 需求背景
当前镜像构建流水线仅产出 linux/amd64 镜像，无法覆盖 arm64 运行环境。

## 变更内容
1. 将镜像构建平台扩展为 linux/amd64 + linux/arm64
2. 保持既有标签/推送策略不变

## 影响范围
- **模块:** config
- **文件:** .github/workflows/build-image.yml
- **API:** 无
- **数据:** 无

## 核心场景

### 需求: 多架构镜像构建
**模块:** config
流水线在推送或手动触发时输出包含 amd64 与 arm64 的镜像清单。

#### 场景: 发布构建
当触发构建时，应同时生成 linux/amd64 与 linux/arm64 镜像并推送到镜像仓库。
- 产出包含双架构的镜像
- 原有标签策略保持一致

## 风险评估
- **风险:** 构建时间增加，可能增加流水线耗时
- **缓解:** 仅修改平台配置，沿用现有 Buildx 与 QEMU 设置
